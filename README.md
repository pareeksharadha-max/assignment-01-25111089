# Data Analytics & Database Management Assignment
**Student ID:** 25111089

## Project Overview
This repository contains a multi-part assignment focused on end-to-end data management, including RDBMS schema design, NoSQL querying, Data Warehousing (Star Schema), and Data Lake architecture.

## Repository Structure
* **datasets/**: Raw data files (CSV, JSON, Parquet).
* **part1-rdbms/**: SQL schema design and relational queries.
* **part2-nosql/**: MongoDB query scripts and document samples.
* **part3-datawarehouse/**: Star schema design and ETL documentation.
* **part5-datalake/**: DuckDB queries and architectural trade-off analysis.

## How to Run
1. **SQL:** The scripts in `part1-rdbms` are compatible with PostgreSQL/MySQL.
2. **NoSQL:** Use MongoDB Compass or Shell to execute scripts in `part2-nosql`.
3. **Data Lake:** DuckDB queries in `part5-datalake` can be run in a Python environment or DuckDB CLI.

## Trade-offs & Decisions
Detailed architectural decisions and design trade-offs are documented within the `.md` files in each respective directory.
